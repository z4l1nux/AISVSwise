# C2.7 Multi-Modal Input Validation

> **Parent:** [C02: User Input Validation](C02-User-Input-Validation)

## Purpose

Modern AI systems accept inputs beyond text -- images, audio, video, PDFs, spreadsheets, and other file types. Each modality introduces unique attack surfaces: images can contain steganographic payloads or adversarial perturbations invisible to humans; audio can include ultrasonic commands; documents can embed hidden instructions in metadata, comments, or invisible layers. Multi-modal validation must handle format verification, content extraction, malware scanning, and adversarial detection for each modality, while also detecting coordinated cross-modal attacks where benign inputs in individual modalities combine to form an attack.

As of March 2026, multi-modal attacks have matured significantly. Transfer-based adversarial attacks achieve concerning success rates against production vision-language models (GPT-4o, Claude 3.5, Gemini). Research published at CVPR 2025 and ICLR 2025 demonstrates chain-of-attack techniques against VLMs, and a 2025 practitioner survey (arXiv:2505.03084) confirms that defense literature remains fragmented with no unified multi-modal defense framework in production.

A January 2026 CSA research note quantifies image-based prompt injection success rates: typographic text injection achieves 64% success against GPT-4V, Claude 3, and Gemini; steganographic encoding reaches 24.3% with near-invisible pixel-level payloads; and the CrossInject framework (ACM MM 2025) improves adversarial perturbation success by at least 30.1% over prior methods through Visual Latent Alignment and Textual Guidance Enhancement using surrogate open-source LLMs. On the defense side, the Cross-Agent Multimodal Provenance-Aware Defense Framework (arXiv:2512.23557) demonstrates 94% detection accuracy compared to 52% for keyword filtering and 66% for safety fine-tuning alone, using cooperating sanitizer agents with a provenance ledger. DMAST (Dual-Modality Multi-Stage Adversarial Safety Training, March 2026) reduces attacker success on multimodal web agents from 41.2% to 21.4% through three-stage adversarial co-evolution training.

## Implementation Maturity (as of March 2026)

| Capability | Maturity Level | Notes |
|------------|---------------|-------|
| File type/size/format validation | High | Standard web application security practice; well-supported by libraries |
| MIME type validation via magic bytes | High | `python-magic`, `file` command; straightforward to implement |
| Metadata extraction and screening | Medium-High | Libraries exist for all common formats; gap is routing extracted text through injection detection |
| Malware scanning in upload pipeline | High | ClamAV, commercial AV, VirusTotal API all production-ready |
| Active content removal (macros, scripts) | Medium-High | Well-supported for Office, PDF, SVG; less mature for newer formats |
| Steganography detection | Low-Medium | Tools exist but have high false positive rates; cannot detect all methods |

| Adversarial perturbation detection (audio) | Low | Research-stage; limited production tooling for ASR adversarial detection |

## Open Research Questions

- Can cross-modal adversarial attacks be reliably detected with current tooling, or is this fundamentally a research problem? As of March 2026, the Provenance-Aware Defense Framework (94% detection) and DMAST (41.2% to 21.4% attacker success reduction) suggest detection is becoming feasible, but these remain research prototypes requiring custom integration -- no turnkey production solution exists yet.
- How should multi-modal validation handle new modalities (e.g., 3D models, haptic input, sensor data) as AI systems expand?
- What is the performance overhead of comprehensive multi-modal validation, and how does it impact real-time applications? CrowdStrike AIDR demonstrates sub-30ms is achievable for text-based detection, but multi-modal scanning adds significant latency.
- How do unified vision-language model architectures (e.g., GPT-4o, Gemini 2.5 Pro) change the attack surface compared to pipeline architectures with separate vision and language models? CVPR 2025 research on chain-of-attack techniques suggests unified architectures may be more susceptible to cascading cross-modal attacks.
- What are effective defenses against typographic attacks on vision models beyond text detection in images? MAT (2025) shows promise at training time, but runtime defenses remain limited.
- How should organizations prioritize adversarial detection investment given that embedding-space attacks are particularly difficult to detect even for experienced practitioners?
- What role should the eight-stage pipeline defense framework (ingestion through monitoring) play in standardized multi-modal validation architectures?
- How should organizations balance the DMAST finding that visual attacks (34.4% success) significantly outperform text-only injections (24.1%), given that most current safety training remains text-centric?
- What is the practical overhead and latency impact of multi-agent sanitization architectures (Text Sanitizer + Visual Sanitizer + Output Validator) compared to single-model defense approaches?
