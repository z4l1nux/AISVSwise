# C7.8 Source Attribution & Citation Integrity

[Back to C07 Index](C07-Model-Behavior)

## Purpose

Source attribution and citation integrity controls ensure that when a system uses retrieval-augmented generation (RAG) to ground its responses in source documents, the citations are genuine and verifiable. LLMs are known to fabricate citations — generating plausible-looking but nonexistent references, or citing real sources that do not actually support the claimed content. In RAG architectures, citations should be derived from retrieval metadata (which documents were actually retrieved) rather than generated by the model (which can hallucinate sources). This section addresses both attribution (linking responses to retrieved documents) and verification (confirming cited claims are actually supported by the cited source).

### 2024-2026 Research: Mechanistic Detection, Hybrid Retrieval, and Multi-Evidence Verification

Research into citation hallucination and grounded generation has advanced substantially since 2024, with several key developments relevant to implementing these requirements.

**Understanding hallucination sources in RAG.** A systematic review of hallucination in retrieval-augmented LLMs (Hallucination Mitigation for Retrieval-Augmented Large Language Models, 2025) identifies two primary failure stages: (1) retrieval failure -- encompassing data source quality, query formulation, retriever limitations, and retrieval strategy problems; and (2) generation deficiency -- including context noise, context conflict, the "lost in the middle" curse (where models underweight information in the middle of long contexts), alignment problems, and capability boundaries. This taxonomy is directly useful for designing verification controls: different failure modes require different detection strategies.

**Mechanistic citation hallucination detection (FACTUM).** The FACTUM framework (January 2026) represents a significant advance by reframing citation hallucination as a "coordination failure between the Attention (reading) and Feed-Forward Network (recalling) pathways" inside transformer models. FACTUM uses four mechanistic scores -- Contextual Alignment Score, Attention Sink Usage, Parametric Force, and Pathway Alignment -- to detect problematic citations by examining the model's internal states during generation. It outperforms state-of-the-art baselines by up to 37.5% in AUC for citation hallucination detection. Critically, the research found that correctness signatures evolve with model scale: 3B-parameter models rely on high pathway alignment, while 8B-parameter models employ a specialized strategy where pathways provide distinct, orthogonal information. This scale-dependent behavior means detection strategies cannot be one-size-fits-all.

**Hybrid retrieval for citation grounding.** A December 2025 study on citation-grounded code comprehension demonstrated that hybrid fusion retrieval (combining multiple retrieval modes with optimized parameters) achieved 92% citation accuracy across all tested models, representing a 14-18 percentage point improvement over single-mode retrieval baselines. The approach requires LLMs to cite specific line ranges that must overlap with retrieved chunks, enforced through interval arithmetic -- a concrete, verifiable citation mechanism.

**Multi-evidence guided answer refinement.** The MEGA-RAG framework (2025) addresses hallucination in high-stakes domains (public health) through multi-evidence guided answer refinement, where multiple retrieved evidence passages are cross-checked before the final response is composed. This approach aligns with requirement 7.8.2's goal of ensuring claims are actually supported by retrieved content.

**Agentic and tool-augmented verification.** A comprehensive survey on hallucination mitigation (2025) highlights the emergence of agentic systems that incorporate retrieval verification as an explicit reasoning step -- the model is prompted or fine-tuned to verify its own citations against retrieved context before emitting a response, sometimes using separate verification tool calls. This connects to AISVS C7.2.5's requirement for tracking tool invocation history as a hallucination detection signal.

## Open Research Questions

- How should attribution be presented for responses that synthesize information across multiple retrieved chunks? Should every sentence be individually attributed? MEGA-RAG's multi-evidence approach suggests cross-checking multiple sources is feasible, but the UX for multi-source attribution remains an open design question.
- What entailment threshold is appropriate for flagging unsupported claims -- strict logical entailment, or a softer "consistent with" standard? NLI models and RAGAS faithfulness metrics use different thresholds, and the optimal choice likely varies by domain risk level.
- How can claim-level verification be made fast enough for real-time serving without unacceptable latency? FACTUM's mechanistic approach (examining model internals during generation) may be faster than post-hoc NLI checking, but requires access to model activations that may not be available with API-based LLMs.
- Given FACTUM's finding that citation correctness signatures are scale-dependent (3B vs. 8B models behave differently), how should verification strategies adapt across different model sizes and architectures? Is a universal detection approach feasible, or must verification be calibrated per model?
- How should systems handle partial support -- where a retrieved chunk supports part of a claim but not all of it? The hybrid retrieval approach of requiring overlap between cited line ranges and retrieved chunks (interval arithmetic verification) offers one concrete mechanism, but generalization beyond code to natural language remains open.
- How do retrieval-failure vs. generation-deficiency hallucination modes (as taxonomized in 2025 review literature) map to different verification strategies? Should the system detect which failure mode is occurring and apply targeted mitigation?
- Should users be able to click through citations to view the original source chunk, and what are the security implications of exposing retrieval internals?
- As agentic RAG systems emerge (where the model can iteratively retrieve, verify, and refine), how should citation integrity controls adapt to multi-step retrieval workflows where the final response synthesizes across multiple retrieval rounds?
